# DISCORD.md

Este projeto utiliza Discord como interface principal.

Canal oficial: #quemresolvebackend

## Regras

1. Apenas o Product Agent interpreta novas ideias.
2. Developer nunca aceita tarefas diretamente do usuário.
3. Reviewer nunca discute código fora de PR.
4. Lead pode responder status e relatórios.

## Fluxo padrão

Usuário escreve no canal → Product interpreta → Issue criada → Developer implementa → Reviewer revisa → Lead reporta status.

## Nunca

- Criar tarefa só no Discord
- Tomar decisão técnica sem Issue
- Resolver conflito fora do GitHub
