# AGENTS.md

Este projeto opera com múltiplos agentes especializados.

## Regras Globais

1. Toda nova demanda começa com o Product Agent.
2. Nenhum Developer cria Issue.
3. Nenhum código é entregue fora de PR.
4. Reviewer é responsável por qualidade.
5. Lead supervisiona prioridades.

## Fluxo

User → Product → Issue GitHub → Developer → PR → Reviewer → Merge
